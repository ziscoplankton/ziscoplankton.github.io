OSI Model

The Open Systems Interconnection model is a conceptual model used in computer science to describe the processes that computer systems use to communicate over a network. It is composed of seven layers:

  • Application
  • Presentation
  • Session
  • Transport
  • Network
  • Data link
  • Physical

These model is usually divided in two parts, the first 4 layers are associated to networking and the last three to development. Combined it is also called DevOps. The model goes from top to bottom in the de-encapsulation process but going from bottom to top in the encapsulation process which can be named sending data and receiving data.

Data Structures

From arrays to tries, data structures can be very useful when it comes to storage and data access, via different algorithms such as linear search, binary search,... aiming for the quickest results.
The simpliest data structure would be an array. It is a contiguous portion of memory addresses that are holding data of a unique data type.
Therefore, a more sophisticated way to store data is to dynamically allocate memory addresses and create structures by linking these memory blocks together. These data structures can be very interesting as it can hold multiple data types and values but cheeky to create without loosing track of their head or breaking their chaines and tail.

This could be a representation of a data structure where the links are made via a second field within the data structure and each node represent a memory allocation of type 'structure' to create a linked list.

Data Link

Data Link is based on the Layer 2 of the OSI Model . It is known for MAC Addressing and its unit is the Ethernet frame. This unit includes layers itself:

  1. Preamble
    A series of 7 bytes - 56 bites of "10" to create a bit level sync
  2. SFD (Start Frame Delimiter)
    Used to delimit the start of a (new) frame
  3. MAC DST
  4. MAC SRC
  5. 802.1Q (also named dot1q & optional)
    Used in VLANs and trunking
  6. Length
  7. Payload
  8. FCS (Frame Check Sequence)

MAC addresses are written in hex, are 6 bytes and a colon is inserted between pairs:
00:EE:AF:FF:EA:00
This example shows a pattern from the concept of IPv6 EUI-64 which is a method that creates automatically a unique IPv6 addresse using the MAC ADDR and the INT ID(interface). This method inserts "FFFE" between the two halves of a MAC.

MAC addresses halves are actually meaningful in other ways. The first half is usually associated to a constructor and the other is used to uniquely identify a machine onto its hardware.
Which is a why they are also known as BIA ADRR for "Burn In Address". Therefore, frames can be forged.
An 'old classic' is exploiting the Address Resolution Protocol known as ARP spoofing or ARP poisoning.
Another way to mislead communications at this layer is a vlan leak by exploiting vlan knowledges unto dot1q and the native vlans.
Therefore, features on switches can mitigate vulnerabilities by enabling port-security and DAI (Dynamic ARP Inspection).
